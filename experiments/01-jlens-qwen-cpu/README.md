# Experiment 01 -- Jacobian lens on Qwen3.5-0.8B (CPU)

**Status: done (2026-07-07), independently verified.** Anthropic's `jlens` run on the
smallest model with a published pre-fitted lens, on a plain CPU (WSL2, no GPU). One clear
positive result, one clear negative result.

## Setup

- Model: `Qwen/Qwen3.5-0.8B` (24 layers, d_model=1024), loaded in **bf16** to fit ~7.6 GB RAM
- Lens: **pre-fitted**, from HF [`neuronpedia/jacobian-lens`](https://huggingface.co/neuronpedia/jacobian-lens),
  revision `qwen-n1000`, file `qwen3.5-0.8b/jlens/Salesforce-wikitext/Qwen3.5-0.8B_jacobian_lens.pt`
  (48 MB; fitted by Neuronpedia on Salesforce/wikitext, 233 prompts)
- Stack: Python 3.12.3, torch 2.12.1+cpu, transformers 5.13.0, jlens 0.1.0
- `demo.py` reads out three multi-hop factual prompts at layers [6, 12, 18, 22], comparing
  **J-lens vs plain logit-lens vs the model's actual next-token distribution**, then scans
  all layers at the descriptor token for the hidden intermediate concept
- `results.md` is the raw captured output (regenerated by every run)

## Reproduce

```bash
cd experiments/01-jlens-qwen-cpu
python3 -m venv .venv && . .venv/bin/activate
pip install torch --index-url https://download.pytorch.org/whl/cpu   # skip if you have CUDA
pip install -r requirements.txt
python demo.py          # ~1 min on CPU after downloads; rewrites results.md
```

First run downloads ~1.7 GB (model + lens) into `~/.cache/huggingface`.

## Result 1 -- the lens works, and clearly beats the logit-lens baseline

At mid layers the plain logit-lens readout is incoherent multilingual noise while the J-lens
is already on-topic. From `results.md`, on *"the currency used in the country shaped like a
boot is"* (Japan in context):

```
L 6 J-lens    : ' coins'  ' bark'  ' canvas'  ' barrel'  ' coin'  ...
L 6 logit-lens: 'has'  ' ____'  'ieg'  '____'  'imo'  '___'  '匿'  ...   <- noise
L22 J-lens    : ' currency'  ' yen'  ' Yen'  ' currencies'  ' Japanese'  ...
MODEL (L23)   : ' the'  ' '  ' Japanese'  ' not'  ' J'  ' yen'  ...
```

By the penultimate layer (L22 of 24) the J-lens top-10 contains the actual answer tokens on
all three prompts (` yen`, ` euro`/` franc`, ` Atlantic`) -- the answer is visibly forming
inside the network before the model emits anything.

## Result 2 -- the paper's hidden-workspace effect does NOT reproduce at 0.8B

The paper's flagship demo reads out a hidden *intermediate* concept at the descriptor token
(e.g. *Italy* at the word "boot", though Italy never appears in the text). At 0.8B the
intermediate ranks in the hundreds to thousands; the lens mostly shows synonyms of the
descriptor token itself (boots/heel/shoe...):

| Descriptor token | Hidden intermediate | Best lens rank (any layer) |
|------------------|---------------------|----------------------------|
| ` boot` | Italy | 5792 (L15) |
| ` Tower` | France | 100 (L21) |
| ` Carnival` | Brazil | 338 (L12) |

Expected, not a bug: Anthropic's multi-hop demos use 4B+ models and the strong J-space claims
are only established for Claude. Workspace-like structure looks capability-dependent -- which
is the interesting takeaway, and what [experiment 02](../../PLANS.md) is designed to measure.

## Verification

An independent adversarial re-run reproduced `results.md` **byte-for-byte** (only wall-clock
timing lines differed), confirmed the lens checkpoint provenance on disk, and code-reviewed
the readout path (forward pass -> Jacobian transport -> unembed -> top-k; no canned outputs).

## Caveats

- One run, three prompts -- a demo, not an evaluation.
- The lens was fitted on wikitext by a third party (Neuronpedia), not by Anthropic.
- `jlens` upstream is an unmaintained reference implementation.
