# Experiment 02 — scale sweep: where does the hidden-intermediate effect emerge?

**Status: CPU half done (2026-07-07)** — pythia-70m, gpt2, Qwen3.5-0.8B, Qwen3-1.7B, all 93
prompts each. Gemma (license-gated) and Qwen 4B+ (GPU) pending — the harness takes them
with the same command.

## Question

Experiment 01 found that at 0.8B the Jacobian lens can watch the *answer* form, but mostly
cannot read out the *hidden intermediate* of a multi-hop prompt (Italy in "the currency of
the country shaped like a boot"). Anthropic's paper demonstrates that effect on Claude-scale
models. **At what size does it appear?**

## Method

- **Prompts:** all 93 items of Anthropic's `lens-eval-multihop` set (copied here as
  `multihop.json`, Apache-2.0, from
  [anthropics/jacobian-lens](https://github.com/anthropics/jacobian-lens)
  `data/evaluations/`). Each item: a prompt, the unstated intermediate concept(s), and the
  expected answer.
- **Models:** every size with a pre-fitted lens on
  [`neuronpedia/jacobian-lens`](https://huggingface.co/neuronpedia/jacobian-lens) that fits
  the machine at hand — see the registry in `sweep.py` (`python sweep.py --list`).
- **Per prompt, four numbers** (rank 1 = top pick):
  1. **task competence** — rank of the expected answer in the model's real next-token
     distribution. If the model can't do the task, there's no reason to expect a workspace
     trace of it.
  2. **answer preview** — best lens rank of the answer at the last prompt token over all
     fitted layers (exp 01's positive result, now measured across scale).
  3. **hidden intermediate** — best lens rank of the intermediate over **all layers and all
     prompt positions**, excluding positions where the intermediate literally appears. This
     is deliberately generous (an upper bound): if even the best cell of the layer×position
     grid ranks the intermediate poorly, the effect is absent at that scale.
  4. **shuffled control** — the identical layers×positions search for an intermediate
     borrowed from a *different* prompt (deterministic offset; never overlapping the
     prompt's own intermediates or text). Best-over-a-grid metrics have a real noise
     floor — a 70M model that solves 0% of the tasks still put the "intermediate" in its
     top-10 a third of the time before we added this. The signal is `inter` minus
     `control`, not `inter` alone.
- Words are matched by the leading token of their surface forms (space-prefixed always;
  bare/case variants only when the leading token is a solid prefix — see
  `variant_token_ids`). Every record carries a `model_max_ties` counter after we found that
  **bf16 collapses pythia-70m's logits into thousands of exact ties** — small models run in
  fp32 (see the registry), bf16 only where RAM demands it.

## Reproduce

```bash
cd experiments/02-scale-sweep
python3 -m venv .venv && . .venv/bin/activate
pip install torch --index-url https://download.pytorch.org/whl/cpu   # skip if you have CUDA
pip install -r requirements.txt

python sweep.py --list                 # registry + what fits where
python sweep.py --model pythia-70m    # ~1 min CPU
python sweep.py --model gpt2          # ~2 min CPU
python sweep.py --model qwen3.5-0.8b  # ~15 min CPU
python sweep.py --model qwen3-1.7b    # ~1 h CPU, borderline on 7.6 GB RAM
python report.py                       # rebuilds results.md from raw/*.jsonl
```

Runs append to `raw/<model>.jsonl` and are resumable (re-run the same command after an
interruption). To extend the curve: `gemma-3-270m` / `gemma-3-1b` after `hf auth login` +
accepting the Gemma license; `qwen3-4b` / `-8b` / `-14b` on a rented GPU, then copy the
`raw/*.jsonl` files back and re-run `report.py`.

## Results (CPU sizes)

Full tables in [results.md](results.md) (build artifact — regenerated by `report.py`).
Ranks are 1-based; "≤10" = fraction of the 93 prompts where the best rank is 10 or better.

| model | params | task solved (answer in model top-10) | inter ≤10 | control ≤10 |
|---|---|---|---|---|
| pythia-70m | 70M | 0% | 32% | 8% |
| gpt2 | 124M | 4% | 31% | 1% |
| qwen3.5-0.8b | 752M | 41% | **84%** | 31% |
| qwen3-1.7b | 1.7B | 58% | **91%** | 30% |

Four findings:

1. **Intermediate visibility rises steeply to 0.8B and then saturates** (on this
   best-anywhere metric): median best rank 27 → 50 → 2 → 2 across the four sizes, with the
   shuffled control flat around rank 141–223 (small models) / 28–35 (Qwen). The
   inter-minus-control gap grows from ~24 points (70M) to ~61 points (1.7B).
2. **Visibility does not track competence.** Prompts the model solves and prompts it fails
   show *identical* intermediate visibility at every size (median rank 2 vs 2 at both Qwen
   sizes; pythia-70m "sees" intermediates on 32% of prompts while solving zero). Whatever
   the lens is reading, the model doesn't visibly need it to answer correctly.
3. **It surfaces in the wrong place for a workspace story.** Top-10 hits cluster at
   end-of-prompt function tokens (the trailing space, ` is`, ` in`, quotes) and in the
   back half of the network (0.8B: 55/78 hits at ≥50% depth; 1.7B: 73/85) — i.e. the model
   *re-derives* the intermediate en route to the answer. Anthropic's Claude demo reads the
   intermediate at the *descriptor token* ("boot", "Carnival") in mid layers, parked well
   before the answer. This reconciles experiment 01: at the descriptor token
   Brazil@`Carnival` ranked 338 (exp 01), but searched anywhere it ranks 3 (at ` where`,
   L21).
4. **The answer-preview effect scales smoothly** (exp 01's positive result): the answer is
   in the lens's top-10 at the last prompt token for 0% / 2% / 59% / 78% of prompts, and
   at its best layer the lens often ranks the answer better than the model's own final
   distribution (median 3 vs 6 at 1.7B) — final logits get diluted by formatting tokens.

**Reading:** at ≤2B, what the lens exposes looks like *association / late retrieval*, not a
reportable workspace — the concept is readable, but not where the paper's effect lives
(descriptor token, mid-network) and not in a way that predicts task success. Whether the
descriptor-token effect switches on at 4B+ is exactly what the GPU half of this sweep is
for.

## Caveats

- The intermediate metric is an upper bound: best cell over ~30 positions × all layers per
  prompt gives rare low ranks by chance; compare *distributions* across models, not single
  prompts.
- Ranks live on different vocab sizes (GPT-2 50k, Qwen 151k); the ≤10 / ≤100 fractions are
  the more comparable columns.
- The lenses were fitted by Neuronpedia on wikitext — corpus sensitivity is experiment 03's
  question.
- Models span families *and* sizes (pythia/gpt2/qwen at the small end), so the small-model
  points mix "scale" with "family"; the within-Qwen column (0.8B → 1.7B → 4B+…) is the clean
  reading of the curve.
