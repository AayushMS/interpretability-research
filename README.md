# jlens-qwen-experiment

Anthropic's **Jacobian lens** tried out on the smallest model it can realistically run on:
**Qwen/Qwen3.5-0.8B**, on a plain CPU (WSL2, no GPU), using a pre-fitted lens published on
the HuggingFace Hub. One script, one afternoon, one clear positive result and one clear
negative result.

## Background

In July 2026 Anthropic published
[*Verbalizable Representations Form a Global Workspace in Language Models*](https://transformer-circuits.pub/2026/workspace/index.html)
([announcement](https://www.anthropic.com/research/global-workspace)), claiming Claude models
contain a sparse "J-space" that behaves like a global workspace: reportable, steerable, and
causally involved in multi-step reasoning. The measurement tool behind it, the **Jacobian
lens**, was open-sourced at
[anthropics/jacobian-lens](https://github.com/anthropics/jacobian-lens). It transports a
hidden-state vector from any layer into the final-layer basis via a fitted Jacobian and
decodes it through the unembedding matrix -- answering *"what is this internal activation
disposed to make the model say?"*

The paper's results are on Claude (Sonnet/Haiku/Opus 4.5). This repo asks a smaller question:
**what does the lens show on a 0.8B open-weights model?**

## What's here

| File | What it is |
|------|------------|
| `demo.py` | End-to-end demo: loads Qwen3.5-0.8B in bf16 on CPU, pulls the pre-fitted lens from the Hub, reads out three multi-hop factual prompts at several layers. Compares J-lens vs plain logit-lens vs the model's real next-token distribution. |
| `results.md` | Raw captured output of a full run (regenerated by every `demo.py` run). |
| `requirements.txt` | `torch`, `transformers`, and `jlens` straight from Anthropic's repo. |

The lens checkpoint comes from
[`neuronpedia/jacobian-lens`](https://huggingface.co/neuronpedia/jacobian-lens)
(revision `qwen-n1000`), fitted by Neuronpedia on Salesforce/wikitext with 233 prompts.
The same Hub repo ships lenses for Qwen3 1.7B-32B and Qwen3.5/3.6 up to 27B if you have the
hardware for a bigger run.

## Quickstart

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install torch --index-url https://download.pytorch.org/whl/cpu   # skip if you have CUDA
pip install -r requirements.txt
python demo.py          # ~1 min on CPU after downloads; rewrites results.md
```

First run downloads ~1.7 GB (model + 48 MB lens) into `~/.cache/huggingface`.
Needs roughly 4 GB of free RAM (the model is loaded in bf16 for exactly this reason).

## Results

### 1. The lens works -- and clearly beats the logit-lens baseline

At mid layers the plain logit-lens readout is incoherent multilingual noise while the J-lens
is already on-topic. From the actual run (`results.md`), on
*"the currency used in the country shaped like a boot is"* (Japan in context):

```
L 6 J-lens    : ' coins'  ' bark'  ' canvas'  ' barrel'  ' coin'  ...
L 6 logit-lens: 'has'  ' ____'  'ieg'  '____'  'imo'  '___'  '匿'  ...   <- noise
L22 J-lens    : ' currency'  ' yen'  ' Yen'  ' currencies'  ' Japanese'  ...
MODEL (L23)   : ' the'  ' '  ' Japanese'  ' not'  ' J'  ' yen'  ...
```

By the penultimate layer (L22 of 24) the J-lens top-10 contains the actual answer tokens on
all three prompts (` yen`, ` euro`/` franc`, ` Atlantic`) -- the answer is visibly forming
inside the network before the model says anything.

### 2. The paper's headline "hidden workspace" effect does NOT reproduce at 0.8B

Anthropic's flagship demo reads out a hidden *intermediate* concept at the descriptor token --
e.g. *Italy* at the word "boot", even though Italy never appears in the text. On this 0.8B
model the intermediate concept ranks in the hundreds to thousands:

| Descriptor token | Hidden intermediate | Best lens rank (any layer) |
|------------------|---------------------|----------------------------|
| ` boot` | Italy | 5792 |
| ` Tower` | France | 100 |
| ` Carnival` | Brazil | 338 |

The lens mostly shows synonyms of the descriptor token itself (boots/heel/shoe...). This is
the expected outcome, not a bug: Anthropic's multi-hop demos use 4B+ models, and the strong
J-space claims are only established for Claude-scale models. Workspace-like structure looks
capability-dependent -- which is itself the interesting takeaway from running this at 0.8B.

## Caveats

- `jlens` is a **reference implementation**: unmaintained, not accepting contributions.
- One run, three prompts, one seed-free greedy readout -- this is a demo, not an evaluation.
- The lens was fitted on wikitext by a third party (Neuronpedia), not by Anthropic.
